# ASCII Dual-Gaussian Surface Renderer

A lightweight, terminal-based 3D visualization of a mathematical surface, rendered entirely in ASCII characters with ANSI color codes.

Inspired by the famous [donut.c](https://www.a1k0n.net/2011/07/20/donut-math.html), this project implements a rotating view of a composite Gaussian function using a Z-buffer algorithm and point-cloud projection.

## üìê The Mathematics

The rendered surface is generated by the superposition of two broad base Gaussian functions and two sharper peaks. The height $z$ at coordinates $(x, y)$ is defined by:

$$
f(x,y) = \underbrace{\exp\left(-\frac{r_1^4}{1000}\right)}_{\text{Base 1}} + \underbrace{\exp\left(-\frac{r_2^4}{1000}\right)}_{\text{Base 2}} + \underbrace{0.1\exp\left(-r_2^4\right)}_{\text{Peak 1}} + \underbrace{0.1\exp\left(-r_1^4\right)}_{\text{Peak 2}}
$$

Where:
$$r_1^2 = (x-4)^2 + (y-4)^2$$
$$r_2^2 = (x+4)^2 + (y+4)^2$$

This creates a distinctive "Twin Peaks" structure with smooth gradients and sharp local maxima.

## üöÄ Features

* **Real-time Rendering**: Calculates and projects points dynamically.
* **Z-Buffering**: Solves the visibility problem to correctly render overlapping surfaces.
* **Color Mapping**: Uses ANSI escape codes to map height ($z$-value) to colors:
    * **<span style="color:red">Red</span>**: High peaks
    * **<span style="color:cyan">Cyan</span>**: Mid-range slopes
    * **<span style="color:blue">Blue</span>**: Base/Low-level areas
* **Pure C**: No external graphics libraries required, depends only on the standard math library.
